• Symptom: One or more cells are white in color

    Diagnosis: The morphological analysis of the corresponding word(s) is not available.

    Action:

    • Open row 'C' to check whether the analysis is present.
    • If the morphological analysis is missing, then goto UPDATE->Morph and provide the correct analysis.
    • Alternately, you can also make additions directly in the $HOME/public_html/scl/SHMT/MT/prog/morph/local_analysis/mo_ana.txt.
    • After making the changes, run 'make' to compile the changes and make them effective.

  • Symptom: Morph analysis is missing

    Diagnosis: The analysis is pruned out

    Action:

    • First check whether the analysis is present in 'B' row. If it is not present, report to ambapradeep@gmail.com
    • If the analysis is present in 'B' row, then check the pratipadika with its gender in $HOME/scl/public_html/SHMT/MT/prog/prune/default_gen.txt.
    • If the pratipadika with the corresponding gender is missing, add to the file, and run 'make'.

  • Symptom: No kaaraka analysis

    Diagnosis: Either the Minion constraint solver has failed or the CLIPS program needs improvement.

    Action:

    • Check whether it has produced all the possible links by placing the mouse over the sentence number column of row 'A', or opening the image in another window.
    • If all the links exist, then the problem is in the 'minion' the constraint solver. Contact ambapradeep@gmail.com along with the sentence.
    • If any of the desired link is missing, go to missing link section to provide the missing link.

  • Symptom: Wrong kaaraka analysis

    Diagnosis: Either the link is not created by the CLIPS program or the desired output is ranked low.

    Action:

    First check whether the desired link exists by bringing the mouse on the 'A' row of the sentence number column.
    • If the link exists, then open all the kaaraka analyses by bringing the mouse on 'L' row of the sentence number column.
      Check whether the desired analysis exists as one of the solutions. If not, report the bug to ambapradeep@gmail.com
    • If the link does not exist, go to missing link.

  • Symptom: A link is missing.

    Diagnosis: System does not have the necessary information.

    Action:

    • First check that the desired morhological analysis exists.
    • If not, then go to missing morph analysis
    • If the desired morphological analysis is present, then check for various relations as mentioned below.
      • Missing karma
        check whether the verb is listed in $HOME/public_html/scl/SHMT/MT/prog/kAraka/list_n/sakarmaka_XAwu_list. If it is not there, add the verb, run 'make'
      • Missing karaNa
        check whether the verb is listed in $HOME/public_html/scl/SHMT/MT/prog/kAraka/list_n/karaNa_XAwu_list. If it is not there, add the verb, run 'make'
      • Missing sampraxAna
        check whether the verb is listed in $HOME/public_html/scl/SHMT/MT/prog/kAraka/list_n/sampraxAna_XAwu_list. If it is not there, add the verb, run 'make'
      • Missing apAxAna
        check whether the verb is listed in $HOME/public_html/scl/SHMT/MT/prog/kAraka/list_n/apAxAna_XAwu_list. If it is not there, add the verb, run 'make'
      • If the verb is dvikarmaka, and only one karma is marked, check whether the verb is listed in $HOME/public_html/scl/SHMT/MT/prog/kAraka/list_n/xvikarmaka_XAwu_list1, or $HOME/public_html/scl/SHMT/MT/prog/kAraka/list_n/xvikarmaka_XAwu_list2. (xvikarmaka_XAwu_list1 has those dvikarmaka verbs whose muKya karma is aBihiwa in karmaNi prayoga, and xvikarmaka_XAwu_list2 has those dvikarmaka verbs whose gONa karma is aBihiwa in karmaNi prayoga.) If it is not there, add the verb, run 'make'
      • Missing kAlAXikaraNa
        check whether the noun is listed in $HOME/public_html/scl/SHMT/MT/prog/kAraka/list_n/kAlAXikaraNa_list If not present, add and run 'make'.

    • Symptom: Meaning is not substituted.

      Diagnosis:

      Dictionary entry needs to be updated.

      Action:

      Get the category(CAT) of the word (noun,pronoun,avy or verb). Check the entry in the corresponding file in $HOME/public_html/scl/SHMT/MT/data/hi/"CAT".txt
      If entry is missing, add new one.
      If entry already exists, report to ambapradeep@gmail.com with the input sentence.

    • Symptom: Hindi meaning is wrong.

      Diagnosis: Polysemous word

      Action:
      • First check that the word is analysed morphologically, and correct morphological analysis is present, after the kaaraka analysis step.
      • If the word is not analysed correctly morphologically, go to morph analysis is missing
      • If the kaaraka analysis is wrong, POSTPONE handling such cases.
      • If the word has correct morph analysis and also has correct kaaraka analysis, then check whether the word is polysemous.
      • If the word is not polysemous, check the meaning in $HOME/public_html/scl/SHMT/MT/data/hi/"CAT".txt where "CAT" is the category(noun/pronoun/avy/verb) of the word.
      • If the word is polysemous, then WSD rules are to be framed. Report such words with input sentence to shukla.preetidev@gmail.com

  • Symptom: Hindi generation is not correct.

    Diagnosis:

    Action:

    First check whether the kaaraka analysis is correct. If not, then go to     wrong kaaraka analysis.
    If the kaaraka analysis is correct, then choose the appropriate one among the following.
    • if 'ne' is missing,
        Check whether the sentence is in the past tense,
        the word which requires 'ne' is marked as 'karwA' (in the row 'E'), and
        the verb is not listed in the $HOME/public_html/scl/SHMT/MT/data/hi/ne_excptnl_verbs.txt.
      • If the verb is listed in this list, remove it and run 'make' to compile.
      • If the word is not marked as 'karwA', that means the kaaraka analysis is not correct. Go to wrong kaaraka analysis.
    • The declension of viSeRaNa is not correct.
        Check that the word is marked as 'viSeRaNa' by opening row 'E'.
      • If it is not marked as viSeRaNa, then go to wrong kaaraka analysis.
      • If it is marked as a viSeRaNa, then open row 'K' and check whether it has received correct parameters for generation.
      • If not, the agreement module is defective.
      • If the parameters are correct, then go to wrong Hindi generation.
    • The agreement of RaRTI viBakwi is not corre ct.
        Check whether the kaaraka analysis is correct.
      • If the relation is marked as RaRTI sambanXa then check whether it is related to the correct word. If not, the kaaraka analysis is not correct.
      • If the relation is marked as RaRTI sambanXa, and the word is related to the correct word, check the gender of the word to which it is related.
      • If the gender is feminine, while it should be masculine, remove the word from $HOME/public_html/scl/SHMT/MT/data/hi/fem_hnd_noun.lst, and run 'make'.
    • Agreement of the verb is not correct .
        Check whether the kaaraka analysis is correct.
      • If it agrees with karwA (or karma), note if it is feminine.
      • If yes, check whether it is present in $HOME/public_html/scl/SHMT/MT/data/hi/fem_hnd_noun.lst. If not, add and then run 'make'.
      • If the karwA (or karma) is not feminine, then the program needs to be checked. Check which program has gone wrong by opening $HOME/public_html/scl/SHMT/DEMO/tmp_in$$/debug_hindi_gen. Report the bug to ambapradeep@gmail.com with the input sentence and the nature of the bug.
    • The declension of the word is not correct.
      If it is a noun, check the paradigm of the word in the Hindi generator. To check the current paradigm, run the following program: $HOME/public_html/scl/SHMT/MT/prog/hnd_gen/test/get_pdgm.pl If it is a verb, report to ambapradeep@gmail.com along with the input sentence and the expected verb conjugation.
    • ViSeRaNa has an extra vibhakti.
      If the kaaraka analysis is correct, this should not happen. A SERIOUS bug. Report with the input sentence.
